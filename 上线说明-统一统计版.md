# 合作伙伴墙—工业软件品牌展厅：统一统计版上线说明

## 上传到 GitHub 的文件

进入 `partner-wall-showroom` 文件夹，把以下内容上传到 GitHub 仓库根目录：

```text
index.html
admin.html
assets/
api/
package.json
上线说明-统一统计版.md
```

不要只上传 `index.html`、`admin.html`、`assets/`，因为统一统计需要 `api/` 后端接口。

## Vercel 部署

1. 在 Vercel 导入 GitHub 仓库。
2. `Framework Preset` 选择 `Other`。
3. 如果仓库根目录直接有 `index.html`，`Root Directory` 留空。
4. 点击 `Deploy`。

部署完成后，页面可以打开，但此时如果还没配置 KV 数据库，后台会显示“本地演示数据”。

## 开启所有观众统一统计

需要在 Vercel 中创建 KV / Redis 数据库，并把环境变量绑定到项目。

### 方式一：Vercel Storage / KV

1. 打开 Vercel 项目。
2. 进入 `Storage`。
3. 点击 `Create Database`。
4. 选择 `KV` 或 Redis 类型数据库。
5. 按页面提示连接到当前项目。
6. Vercel 会自动生成环境变量，通常包括：

```text
KV_REST_API_URL
KV_REST_API_TOKEN
```

7. 回到项目页面，点击 `Deployments`。
8. 找到最新部署，点击右侧 `...`。
9. 点击 `Redeploy`，让环境变量生效。

### 方式二：Upstash Redis

如果你使用 Upstash Redis，需要在 Vercel 项目中手动添加环境变量：

```text
UPSTASH_REDIS_REST_URL
UPSTASH_REDIS_REST_TOKEN
```

本项目接口同时兼容 `KV_REST_API_URL / KV_REST_API_TOKEN` 和 `UPSTASH_REDIS_REST_URL / UPSTASH_REDIS_REST_TOKEN`。

## 验证是否成功

1. 打开展厅首页：

```text
https://你的项目名.vercel.app/
```

2. 点击某个品牌区域。
3. 点击该品牌右上角 `分享`，复制生成的分享链接。
4. 用另一个浏览器或手机打开该分享链接。
5. 打开数据看板：

```text
https://你的项目名.vercel.app/admin.html
```

如果顶部提示“当前正在显示云端统一统计数据”，说明配置成功。

## 下载数据

在数据看板点击：

- `下载JSON`：下载完整结构化数据
- `下载CSV`：下载表格数据，适合用 Excel 打开

## 当前统计口径

### 品牌区域汇总

- `区域浏览点击数`：品牌区域详情被打开的总次数
- `区域访问人数`：不同访客数，基于浏览器本地访客 ID 去重
- `分享链接打开次数`：所有分享链接被打开的总次数
- `分享触达人数`：所有分享链接触达的不同访客数

### 分享链接明细

- 每点击一次“分享”，会生成一条独立分享链接。
- 每条分享链接都有自己的点击数和人数。

## 注意事项

当前后台没有管理员密码，知道 `admin.html` 地址的人都能查看数据。如果后续正式投放，建议增加管理员登录或访问口令。
